# SnapTotal 手機優先版型重設計畫

## 核心原則
手機畫面寶貴，首屏只放「最常用的操作」。翻轉畫面結構，讓手機一打開就看到「總金額 + 數字鍵盤 / 拍照按鈕」，不需向下捲動。

---

## 手機首屏結構（目標 < 100vh）

```text
┌────────────────────────────┐
│  Logo小圖  SnapTotal  [⚙]  │  ← 極簡 header，一行搞定（~48px）
├────────────────────────────┤
│  💰 總金額 $1,280           │  ← 巨大字體，一眼看到（~80px）
│  預算 $2,000 ▓▓▓▓▓░░░ 64%  │  ← 進度條整合在同一行（~32px）
├────────────────────────────┤
│  [手動輸入]  [拍照辨識]      │  ← Tab 切換（~48px）
├────────────────────────────┤
│  品名(可不填)  金額顯示 $75  │  ← 合併一行（~48px）
│  ┌─────┬─────┬─────┐       │
│  │  1  │  2  │  3  │       │  ← 數字鍵盤佔滿剩餘空間
│  │  4  │  5  │  6  │       │    （~280px）
│  │  7  │  8  │  9  │       │
│  │ 00  │  0  │  ←  │       │
│  └─────┴─────┴─────┘       │
│  [＋ 加入清單]               │  ← 全寬主要按鈕（~56px）
├────────────────────────────┤
│  購物清單 (3筆) ∨           │  ← 可收合，預設收合（~48px）
└────────────────────────────┘
```

---

## 實作步驟 (Phases)

### Phase A — Header 與總金額壓縮
1. 將 `hero-card` 在手機上改為極簡 sticky header：Logo 小圖（32px）+ 「SnapTotal」文字 + 設定齒輪 icon，只佔一行。
2. 三張 `summary-card` 壓縮成一個「總金額大字 + 預算進度條」的緊湊區塊。
3. 預算設定輸入和長輩模式 toggle 移到點擊齒輪 icon 才會顯示的 Modal 或 Dialog 裡面（桌面版可維持或統一行為）。

### Phase B — 輸入區域最大化
4. Tab 按鈕保持大字體，但略微壓縮高度（padding 降低）。
5. 手動輸入：品名與金額顯示合併成一行排版，節省垂直空間。
6. 數字鍵盤高度微降，但字體維持大字。
7. 「直接加入」按鈕做成全寬大按鈕，置於數字鍵盤下方。
8. OCR 拍照頁：圖片預覽 `min-height` 降低。

### Phase C — 清單與設定下沉
9. 購物清單在手機上加上可收合的機制（`<details>` 或是依賴 CSS 隱藏），預設在手機上為收起狀態。桌面版（>960px）維持雙欄佈局且始終展開。

### Phase D — 手機專屬 CSS 微調
10. 新增 `@media (max-width: 640px)` 手機專屬規則。
11. 主容器的間距在手機上大幅降低（例如從 `1.25rem` 降到 `0.5rem`）。
12. 加入 `<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />` 支援瀏海型手機。

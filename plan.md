# 📱 購物金額快速累計工具 — 專案計畫書

## 專案概述

開發一款輕量級 Web 工具，讓使用者透過**拍照辨識價格（OCR）**或**手動快速輸入**，即時累計購物金額，幫助家人在超市或賣場購物時輕鬆控管預算。

## 目標使用者

- 主要：家中長輩、不擅長操作複雜 App 的家人
- 次要：任何希望在購物時快速掌握花費的人

## 核心問題

> 逛超市時邊買邊算總金額，心算容易出錯，操作計算機又不方便。

---

## 核心功能

| 功能 | 說明 |
|------|------|
| 📷 拍照辨識價格 | 使用手機鏡頭拍攝價格標籤，透過 OCR 自動辨識金額 |
| ⌨️ 手動快速輸入 | 提供大字數字鍵盤，一鍵輸入金額加入清單 |
| 📋 購物清單 | 即時顯示已加入的品項與金額列表 |
| 💰 即時累計總金額 | 畫面上方始終顯示目前購物總金額 |
| ✏️ 快速修正 | 辨識錯誤或輸入錯誤時，可快速編輯或刪除項目 |
| 🎯 預算提醒 | 可設定預算上限，接近或超過時提示警告 |

## 技術架構

### 前端（主要）

- **框架**：React（或 Vue）搭配 Vite 建置
- **UI 設計**：Mobile-first RWD，大按鈕、大字體，適合長輩操作
- **PWA 支援**：可安裝至手機桌面，離線也能使用手動輸入功能

### OCR 辨識

- **方案 A（前端）**：使用 [Tesseract.js](https://github.com/naptha/tesseract.js) 在瀏覽器端執行 OCR，無需後端
- **方案 B（雲端 API）**：串接 Google Cloud Vision API 或其他 OCR 服務，辨識準確度較高
- **建議**：先以方案 A 為主（零成本、離線可用），後續可視準確度決定是否升級

### 價格擷取邏輯

- OCR 回傳文字後，透過正則表達式（Regex）擷取數字格式的價格
- 過濾規則：排除日期、條碼數字、促銷文字等干擾
- 若辨識到多個數字，提供選擇介面讓使用者確認

### 資料儲存

- 使用 `localStorage` 或 `IndexedDB` 儲存當次購物清單
- 不需要後端資料庫，保持輕量

### 1. 主畫面（首頁）

```
┌──────────────────────────┐
│   💰 目前總金額：$1,280   │
│   📊 預算：$2,000（剩餘 $720）│
```

### 2. 拍照辨識畫面

- 開啟相機 → 拍攝價格標籤 → 顯示辨識結果
- 使用者確認金額（可修改）→ 加入清單

### 3. 手動輸入畫面

- 大字數字鍵盤 + 快速加入按鈕
- 可選填品名（非必填，降低操作門檻）

## 技術挑戰與解法

| 挑戰 | 解法 |
|------|------|
| 📸 OCR 準確度不穩定 | 加入使用者確認步驟；提供快速修正功能 |
| 🧠 價格 vs 非價格數字 | 正則匹配常見價格格式（$XX、NT$XX、XX元）；多結果時讓使用者選擇 |
| ⚡ 操作要比計算機快 | 大按鈕 UI、一鍵加入、減少步驟數 |
| 🔄 錯誤修正 | 清單項目支援滑動刪除、點擊編輯 |
| 👴 長輩友善 | 大字體（至少 18px）、高對比色、簡潔介面 |
